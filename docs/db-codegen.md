# DB Codegen — PostgreSQL Schema → TanStack Query Hook 자동생성

## 개요

PostgreSQL 데이터베이스에 직접 연결하여 테이블 스키마를 조회하고,
TanStack Query 용 **DTO 타입**, **Query Key Factory**, **CRUD Hook**을 자동 생성하는 코드 생성기.

```
PostgreSQL DB
    ↓  (introspection via information_schema)
codegen script (Node.ts)
    ↓  (template → file write)
├── generated/db/types.ts        ← Row / Insert / Update 타입
├── generated/db/queryKeys.ts    ← query key factory
└── generated/db/hooks.ts        ← useXxxList, useInsertXxx, useUpdateXxx, useDeleteXxx
```

---

## 구현 단계 요약

| Phase | 범위 | 상태 |
|-------|------|------|
| **Phase 1** | Codegen 코어 — introspection, 타입/hook 생성, dry-run, 테스트 | ✅ 완료 |
| **Phase 2** | Electron DB 레이어 강화 — 에러 처리, 트랜잭션, bulk insert, CSV import, pool 관리 | 🔲 미구현 |

# Phase 1: Codegen 코어 ✅

## 동기

기존 방식은 SQL 문자열을 직접 작성하고 타입을 수동 정의해야 한다:

```tsx
// Before — 수동 타입 + raw SQL
type SorvRow = Record<string, unknown> & { sorv_path?: string };

const rows = await window.electronApi.dbApi.query<SorvRow>(
  `SELECT * FROM public.sorv_task ORDER BY 1 DESC`,
);
```

자동 생성 후에는 타입 안전한 hook을 바로 사용할 수 있다:

```tsx
// After — 생성된 hook 사용
const { data, isLoading } = useSorvTaskList();
const insert = useInsertSorvTask();
insert.mutate({ sorv_path: '/some/path' });
```

- SQL 문자열 제거
- 타입 안전 보장
- `invalidateQueries` 자동 처리

## 내부 동작

### 1. DB Schema Introspection

`information_schema`를 조회하여 테이블/컬럼 메타데이터를 수집한다:

```sql
SELECT
  c.table_schema, c.table_name, c.column_name, c.udt_name,
  c.is_nullable, c.column_default,
  COALESCE(pk.is_pk, false) AS is_primary_key
FROM information_schema.columns c
JOIN information_schema.tables t ...
LEFT JOIN ( ... PRIMARY KEY subquery ... ) pk ...
WHERE c.table_schema = ANY($1)
ORDER BY c.table_schema, c.table_name, c.ordinal_position;
```

파싱 결과 내부 메타 구조:

```ts
interface TableMeta {
  schema: string;        // "public"
  name: string;          // "sorv_task"
  columns: ColumnMeta[];
}

interface ColumnMeta {
  name: string;          // "id"
  pgType: string;        // "int4", "text", "timestamptz"
  nullable: boolean;
  isPrimaryKey: boolean;
  hasDefault: boolean;   // serial, now() 등
}
```

### 2. PG Type → TS Type 매핑

| PostgreSQL (`udt_name`) | TypeScript |
|--------------------------|------------|
| `int2`, `int4`, `int8`, `float4`, `float8`, `numeric` | `number` |
| `bool` | `boolean` |
| `text`, `varchar`, `char`, `uuid` | `string` |
| `date`, `timestamp`, `timestamptz` | `string` |
| `json`, `jsonb` | `unknown` |
| `_int4`, `_text` 등 배열 | `number[]`, `string[]` 등 |
| 그 외 | `unknown` |

### 3. 타입 생성 규칙

각 테이블마다 3가지 타입을 생성한다:

| 생성 타입 | PK / default 컬럼 | nullable 컬럼 | 나머지 |
|-----------|-------------------|---------------|--------|
| `XxxRow` | required | `T \| null` | required |
| `XxxInsert` | optional (`?`) | optional (`?`) | required |
| `XxxUpdate` | `Partial<XxxInsert>` | — | — |

# Phase 2: Electron DB 레이어 강화 🔲

Phase 1의 codegen은 기존 `window.electronApi.dbApi.query()`를 그대로 사용한다.
Phase 2에서는 Electron main process의 DB 레이어를 강화하여 새로운 IPC 채널을 추가하고,
codegen도 이에 맞게 확장한다.

## 현재 코드의 문제점

```ts
// dbHandlers.ts — 현재
ipcMain.handle('db:query', async (_event, sql: string, params?: any[]) => {
  return await query(sql, params);  // 에러 처리 없음, 단일 API만 존재
});
```

- renderer에서 **임의의 SQL을 직접 전달** → 구조적 결함
- **에러가 raw로 renderer에 전달** → PG 내부 에러 메시지가 사용자에게 노출
- **단일 `query` 채널만 존재** → 트랜잭션, bulk insert, COPY 등 고급 기능 불가
- **connection pool 설정 미노출** → pool exhaustion 시 디버깅 어려움

## Phase 2-1: 구조화된 에러 처리

**파일**: `apps/electron/src/db/postgres.ts`, `apps/electron/src/ipc/dbHandlers.ts`

```ts
// postgres.ts
export interface DbError {
  code: string;        // PG 에러 코드 (e.g. '23505' = unique violation)
  message: string;     // 사용자 친화적 메시지
  detail?: string;
  constraint?: string;
}

function pgErrorToMessage(err: DatabaseError): string {
  switch (err.code) {
    case '23505': return `중복 데이터: ${err.constraint ?? err.detail}`;
    case '23503': return `참조 무결성 위반: ${err.detail}`;
    case '23502': return `필수 값 누락: ${err.column}`;
    case '42P01': return `테이블이 존재하지 않음`;
    case '57014': return `쿼리 타임아웃`;
    default:      return err.message;
  }
}
```

```ts
// dbHandlers.ts — IPC 응답 래핑
ipcMain.handle('db:query', async (_event, sql, params) => {
  try {
    return { ok: true, data: await query(sql, params) };
  } catch (err) {
    return { ok: false, error: err as DbError };
  }
});
```

## Phase 2-2: 트랜잭션 지원

**파일**: `apps/electron/src/db/postgres.ts`, `apps/electron/src/ipc/dbHandlers.ts`, `preload.ts`, `global.d.ts`

```ts
// postgres.ts
export async function transaction<T>(
  fn: (client: PoolClient) => Promise<T>,
): Promise<T> {
  const client = await pool.connect();
  try {
    await client.query('BEGIN');
    const result = await fn(client);
    await client.query('COMMIT');
    return result;
  } catch (err) {
    await client.query('ROLLBACK');
    throw err;
  } finally {
    client.release();
  }
}
```

```ts
// IPC 채널
ipcMain.handle('db:transaction', async (_event, queries: { sql: string; params?: unknown[] }[]) => {
  return await transaction(async (client) => {
    const results = [];
    for (const q of queries) {
      results.push(await client.query(q.sql, q.params));
    }
    return results.map(r => r.rows);
  });
});
```

## Phase 2-3: COPY 기반 Bulk Insert

**파일**: `apps/electron/src/db/postgres.ts`, `apps/electron/src/ipc/dbHandlers.ts`
**추가 의존성**: `pg-copy-streams`

수천~수만 건의 데이터를 한 번에 넣을 때 `INSERT ... VALUES` 반복은 느리다.
PostgreSQL의 `COPY` 프로토콜을 사용하면 **10~100배 빠른** bulk insert가 가능하다.

```ts
// postgres.ts
export async function bulkInsert(
  table: string,
  columns: string[],
  rows: unknown[][],
): Promise<number> {
  const client = await pool.connect();
  try {
    const copySQL = `COPY ${table} (${columns.join(',')}) FROM STDIN WITH (FORMAT csv)`;
    const stream = client.query(copyFrom.from(copySQL));
    const csvLines = rows.map(row =>
      row.map(v => (v == null ? '' : String(v))).join(','),
    );
    const readable = Readable.from(csvLines.map(line => line + '\n'));
    await pipeline(readable, stream);
    return rows.length;
  } finally {
    client.release();
  }
}
```

사용 예시 (renderer):

```ts
await window.electronApi.dbApi.bulkInsert(
  'public.sorv_task',
  ['sorv_path', 'status'],
  paths.map(p => [p, 'pending']),
);
```

## Phase 2-4: CSV 파일 직접 Import

**파일**: `apps/electron/src/db/postgres.ts`, `apps/electron/src/ipc/dbHandlers.ts`

```ts
export async function importCsv(
  table: string,
  columns: string[],
  csvFilePath: string,
): Promise<void> {
  const client = await pool.connect();
  try {
    const copySQL = `COPY ${table} (${columns.join(',')}) FROM STDIN WITH (FORMAT csv, HEADER true)`;
    const stream = client.query(copyFrom.from(copySQL));
    const fileStream = createReadStream(csvFilePath);
    await pipeline(fileStream, stream);
  } finally {
    client.release();
  }
}
```

## Phase 2-5: Connection Pool 관리

**파일**: `apps/electron/src/db/postgres.ts`, `apps/electron/src/main.ts`

```ts
const pool = new Pool({
  connectionString: process.env.DATABASE_URL,
  max: 10,
  idleTimeoutMillis: 30_000,
  connectionTimeoutMillis: 5_000,
  statement_timeout: 30_000,  // 쿼리 타임아웃 30초
});

pool.on('error', (err) => {
  console.error('[DB Pool] Unexpected error on idle client:', err);
});

export async function closePool(): Promise<void> {
  await pool.end();
}
```

앱 lifecycle 연결:

```ts
// main.ts
app.on('before-quit', async () => {
  await closePool();
});
```

Pool 상태 조회 IPC (디버깅용):

```ts
ipcMain.handle('db:pool-status', () => ({
  total: pool.totalCount,
  idle: pool.idleCount,
  waiting: pool.waitingCount,
}));
```

## Phase 2-6: Codegen 확장 — Bulk Insert Hook 생성

Phase 2-3의 `bulkInsert` IPC가 구현된 후,
codegen이 CRUD hook 외에 bulk insert hook도 자동 생성하도록 확장한다:

```ts
// generated/db/hooks.ts — 추가 생성
export function useBulkInsertSorvTask() {
  const qc = useQueryClient();
  return useMutation({
    mutationFn: (rows: SorvTaskInsert[]) =>
      window.electronApi.dbApi.bulkInsert(
        'public.sorv_task',
        ['sorv_path'],
        rows.map(r => [r.sorv_path]),
      ),
    onSuccess: () => qc.invalidateQueries({ queryKey: dbKeys.sorvTask.all }),
  });
}
```
