# DB Codegen — PostgreSQL Schema → TanStack Query Hook 자동생성

## 개요

PostgreSQL 데이터베이스에 직접 연결하여 테이블 스키마를 조회하고,
TanStack Query 용 **DTO 타입**, **Query Key Factory**, **CRUD Hook**을 자동 생성하는 코드 생성기.

```
PostgreSQL DB
    ↓  (introspection via information_schema)
codegen script (Node.ts)
    ↓  (template → file write)
├── generated/db/types.ts        ← Row / Insert / Update 타입
├── generated/db/queryKeys.ts    ← query key factory
└── generated/db/hooks.ts        ← useXxxList, useInsertXxx, useUpdateXxx, useDeleteXxx
```

---

## 동기

기존 방식은 SQL 문자열을 직접 작성하고 타입을 수동 정의해야 한다:

```tsx
// Before — 수동 타입 + raw SQL
type SorvRow = Record<string, unknown> & { sorv_path?: string };

const rows = await window.electronApi.dbApi.query<SorvRow>(
  `SELECT * FROM public.sorv_task ORDER BY 1 DESC`,
);
```

자동 생성 후에는 타입 안전한 hook을 바로 사용할 수 있다:

```tsx
// After — 생성된 hook 사용
const { data, isLoading } = useSorvTaskList();
const insert = useInsertSorvTask();
insert.mutate({ sorv_path: '/some/path' });
```

- SQL 문자열 제거
- 타입 안전 보장
- `invalidateQueries` 자동 처리

## 내부 동작

### 1. DB Schema Introspection

`information_schema`를 조회하여 테이블/컬럼 메타데이터를 수집한다:

```sql
SELECT
  t.table_schema,
  t.table_name,
  c.column_name,
  c.data_type,
  c.udt_name,
  c.is_nullable,
  c.column_default,
  EXISTS (
    SELECT 1 FROM information_schema.key_column_usage k
    JOIN information_schema.table_constraints tc
      ON k.constraint_name = tc.constraint_name
    WHERE tc.constraint_type = 'PRIMARY KEY'
      AND k.table_name = c.table_name
      AND k.column_name = c.column_name
  ) AS is_primary_key
FROM information_schema.tables t
JOIN information_schema.columns c
  ON t.table_name = c.table_name
  AND t.table_schema = c.table_schema
WHERE t.table_schema NOT IN ('pg_catalog', 'information_schema')
  AND t.table_type = 'BASE TABLE'
ORDER BY t.table_schema, t.table_name, c.ordinal_position;
```

파싱 결과 내부 메타 구조:

```ts
interface TableMeta {
  schema: string;        // "public"
  name: string;          // "sorv_task"
  columns: ColumnMeta[];
}

interface ColumnMeta {
  name: string;          // "id"
  pgType: string;        // "int4", "text", "timestamptz"
  nullable: boolean;
  isPrimaryKey: boolean;
  hasDefault: boolean;   // serial, now() 등
}
```

### 2. PG Type → TS Type 매핑

| PostgreSQL (`udt_name`) | TypeScript |
|--------------------------|------------|
| `int2`, `int4`, `int8`, `float4`, `float8`, `numeric` | `number` |
| `bool` | `boolean` |
| `text`, `varchar`, `char`, `uuid` | `string` |
| `date`, `timestamp`, `timestamptz` | `string` |
| `json`, `jsonb` | `unknown` |
| 그 외 | `unknown` |

### 3. 타입 생성 규칙

각 테이블마다 3가지 타입을 생성한다:

| 생성 타입 | PK / default 컬럼 | nullable 컬럼 | 나머지 |
|-----------|-------------------|---------------|--------|
| `XxxRow` | required | `T \| null` | required |
| `XxxInsert` | optional (`?`) | optional (`?`) | required |
| `XxxUpdate` | optional (`?`) | optional (`?`) | optional (`?`) |

```ts
// Auto-generated by codegen-db — DO NOT EDIT

import { useQuery, useMutation, useQueryClient } from '@tanstack/react-query';
import type { SorvTaskRow, SorvTaskInsert, SorvTaskUpdate } from './types';
import { dbKeys } from './queryKeys';

// ── SELECT ALL ──────────────────────────────────────
export function useSorvTaskList() {
  return useQuery({
    queryKey: dbKeys.sorvTask.all,
    queryFn: () =>
      window.electronApi.dbApi.query<SorvTaskRow>(
        'SELECT * FROM public.sorv_task ORDER BY id DESC',
      ),
  });
}

// ── SELECT BY PK ────────────────────────────────────
export function useSorvTaskById(id: number) {
  return useQuery({
    queryKey: dbKeys.sorvTask.byId(id),
    queryFn: async () => {
      const rows = await window.electronApi.dbApi.query<SorvTaskRow>(
        'SELECT * FROM public.sorv_task WHERE id = $1',
        [id],
      );
      return rows[0] ?? null;
    },
    enabled: id != null,
  });
}

// ── INSERT ──────────────────────────────────────────
export function useInsertSorvTask() {
  const qc = useQueryClient();
  return useMutation({
    mutationFn: (input: SorvTaskInsert) =>
      window.electronApi.dbApi.query(
        'INSERT INTO public.sorv_task (sorv_path) VALUES ($1)',
        [input.sorv_path],
      ),
    onSuccess: () => qc.invalidateQueries({ queryKey: dbKeys.sorvTask.all }),
  });
}

// ── UPDATE ──────────────────────────────────────────
export function useUpdateSorvTask() {
  const qc = useQueryClient();
  return useMutation({
    mutationFn: ({ id, ...data }: SorvTaskUpdate & { id: number }) => {
      const cols = Object.keys(data);
      const sets = cols.map((c, i) => `${c} = $${i + 2}`).join(', ');
      return window.electronApi.dbApi.query(
        `UPDATE public.sorv_task SET ${sets} WHERE id = $1`,
        [id, ...Object.values(data)],
      );
    },
    onSuccess: () => qc.invalidateQueries({ queryKey: dbKeys.sorvTask.all }),
  });
}

// ── DELETE ──────────────────────────────────────────
export function useDeleteSorvTask() {
  const qc = useQueryClient();
  return useMutation({
    mutationFn: (id: number) =>
      window.electronApi.dbApi.query(
        'DELETE FROM public.sorv_task WHERE id = $1',
        [id],
      ),
    onSuccess: () => qc.invalidateQueries({ queryKey: dbKeys.sorvTask.all }),
  });
}
```

## Electron Main Process (DB 레이어) 리팩터링 계획

현재 Electron 쪽 DB 레이어는 최소 구현 상태이며, 아래 영역의 강화가 필요하다.

### 현재 코드의 문제점

```ts
// dbHandlers.ts — 현재
ipcMain.handle('db:query', async (_event, sql: string, params?: any[]) => {
  return await query(sql, params);  // 에러 처리 없음, 단일 API만 존재
});
```

- renderer에서 **임의의 SQL을 직접 전달** → SQL Injection 위험 (Electron 내부이므로 낮지만 구조적 결함)
- **에러가 raw로 renderer에 전달** → PG 내부 에러 메시지가 사용자에게 노출될 수 있음
- **단일 `query` 채널만 존재** → 트랜잭션, bulk insert, COPY 등 고급 기능 불가
- **connection pool 설정 미노출** → pool exhaustion 시 디버깅 어려움

### 강화 1: 구조화된 에러 처리

```ts
// postgres.ts — 개선
import { Pool, DatabaseError } from 'pg';

export interface DbResult<T> {
  rows: T[];
  rowCount: number;
}

export interface DbError {
  code: string;        // PG 에러 코드 (e.g. '23505' = unique violation)
  message: string;     // 사용자 친화적 메시지
  detail?: string;     // PG detail
  constraint?: string; // 위반된 constraint 이름
}

export async function query<T>(sql: string, params?: unknown[]): Promise<DbResult<T>> {
  try {
    const result = await pool.query(sql, params);
    return { rows: result.rows, rowCount: result.rowCount ?? 0 };
  } catch (err) {
    if (err instanceof DatabaseError) {
      throw {
        code: err.code,
        message: pgErrorToMessage(err),
        detail: err.detail,
        constraint: err.constraint,
      } satisfies DbError;
    }
    throw { code: 'UNKNOWN', message: String(err) } satisfies DbError;
  }
}

function pgErrorToMessage(err: DatabaseError): string {
  switch (err.code) {
    case '23505': return `중복 데이터: ${err.constraint ?? err.detail}`;
    case '23503': return `참조 무결성 위반: ${err.detail}`;
    case '23502': return `필수 값 누락: ${err.column}`;
    case '42P01': return `테이블이 존재하지 않음`;
    case '57014': return `쿼리 타임아웃`;
    default:      return err.message;
  }
}
```

IPC 핸들러에서도 에러를 구조화하여 전달:

```ts
// dbHandlers.ts — 개선
ipcMain.handle('db:query', async (_event, sql, params) => {
  try {
    return { ok: true, data: await query(sql, params) };
  } catch (err) {
    return { ok: false, error: err as DbError };
  }
});
```

### 강화 2: 트랜잭션 지원

여러 쿼리를 하나의 트랜잭션으로 묶어야 하는 경우가 있다.

```ts
// postgres.ts
export async function transaction<T>(
  fn: (client: PoolClient) => Promise<T>,
): Promise<T> {
  const client = await pool.connect();
  try {
    await client.query('BEGIN');
    const result = await fn(client);
    await client.query('COMMIT');
    return result;
  } catch (err) {
    await client.query('ROLLBACK');
    throw err;
  } finally {
    client.release();
  }
}
```

IPC 채널:

```ts
// dbHandlers.ts
ipcMain.handle('db:transaction', async (_event, queries: { sql: string; params?: unknown[] }[]) => {
  return await transaction(async (client) => {
    const results = [];
    for (const q of queries) {
      results.push(await client.query(q.sql, q.params));
    }
    return results.map(r => r.rows);
  });
});
```

Preload / global.d.ts 추가:

```ts
export interface DbApi {
  query<T = any>(sql: string, params?: any[]): Promise<T[]>;
  transaction(queries: { sql: string; params?: unknown[] }[]): Promise<unknown[][]>;
}
```

### 강화 3: COPY를 이용한 대량 데이터 Bulk Insert

수천~수만 건의 데이터를 한 번에 넣을 때 `INSERT ... VALUES` 반복은 느리다.
PostgreSQL의 `COPY` 프로토콜을 사용하면 **10~100배 빠른** bulk insert가 가능하다.

```ts
// postgres.ts
import { pipeline } from 'stream/promises';
import { Readable } from 'stream';
import copyFrom from 'pg-copy-streams';

export async function bulkInsert(
  table: string,
  columns: string[],
  rows: unknown[][],
): Promise<number> {
  const client = await pool.connect();
  try {
    const copySQL = `COPY ${table} (${columns.join(',')}) FROM STDIN WITH (FORMAT csv)`;
    const stream = client.query(copyFrom.from(copySQL));

    // rows → CSV 텍스트 변환
    const csvLines = rows.map(row =>
      row.map(v => (v == null ? '' : String(v))).join(','),
    );
    const readable = Readable.from(csvLines.map(line => line + '\n'));

    await pipeline(readable, stream);
    return rows.length;
  } finally {
    client.release();
  }
}
```

IPC 채널:

```ts
ipcMain.handle(
  'db:bulk-insert',
  async (_event, table: string, columns: string[], rows: unknown[][]) => {
    return await bulkInsert(table, columns, rows);
  },
);
```

Preload / global.d.ts 추가:

```ts
export interface DbApi {
  query<T = any>(sql: string, params?: any[]): Promise<T[]>;
  transaction(queries: { sql: string; params?: unknown[] }[]): Promise<unknown[][]>;
  bulkInsert(table: string, columns: string[], rows: unknown[][]): Promise<number>;
}
```

사용 예시 (renderer):

```ts
// 10,000건 한 번에 insert
await window.electronApi.dbApi.bulkInsert(
  'public.sorv_task',
  ['sorv_path', 'status'],
  paths.map(p => [p, 'pending']),
);
```

추가 의존성: `pg-copy-streams` (devDependency)

### 강화 4: CSV 파일 직접 Import

대용량 CSV 파일을 읽어서 테이블에 직접 넣는 기능.

```ts
// postgres.ts
import { createReadStream } from 'fs';

export async function importCsv(
  table: string,
  columns: string[],
  csvFilePath: string,
): Promise<void> {
  const client = await pool.connect();
  try {
    const copySQL = `COPY ${table} (${columns.join(',')}) FROM STDIN WITH (FORMAT csv, HEADER true)`;
    const stream = client.query(copyFrom.from(copySQL));
    const fileStream = createReadStream(csvFilePath);
    await pipeline(fileStream, stream);
  } finally {
    client.release();
  }
}
```

IPC 채널:

```ts
ipcMain.handle(
  'db:import-csv',
  async (_event, table: string, columns: string[], filePath: string) => {
    await importCsv(table, columns, filePath);
  },
);
```

### 강화 5: Connection Pool 관리

```ts
// postgres.ts — 개선
const pool = new Pool({
  connectionString: process.env.DATABASE_URL,
  max: 10,                    // 최대 연결 수
  idleTimeoutMillis: 30_000,  // 유휴 연결 타임아웃
  connectionTimeoutMillis: 5_000,
  statement_timeout: 30_000,  // 쿼리 타임아웃 30초
});

// 연결 상태 모니터링
pool.on('error', (err) => {
  console.error('[DB Pool] Unexpected error on idle client:', err);
});

// 앱 종료 시 정리
export async function closePool(): Promise<void> {
  await pool.end();
}
```

앱 lifecycle에 연결:

```ts
// main.ts
app.on('before-quit', async () => {
  await closePool();
});
```

Pool 상태 조회 IPC (디버깅용):

```ts
ipcMain.handle('db:pool-status', () => ({
  total: pool.totalCount,
  idle: pool.idleCount,
  waiting: pool.waitingCount,
}));
```

## 기존 코드와의 관계

이 codegen은 기존 IPC 레이어(`preload.ts` → `dbHandlers.ts` → `postgres.ts`)를 그대로 활용한다.
생성된 hook은 `window.electronApi.dbApi.query()`를 내부적으로 호출하므로,
Electron main process 쪽 변경 없이 renderer 코드만 교체하면 된다.

```
┌─ Renderer ──────────────────────────┐
│  generated/hooks.ts                 │  ← 자동 생성
│    └→ window.electronApi.dbApi.query│
├─ Preload ───────────────────────────┤
│  preload.ts (변경 없음)              │
├─ Main ──────────────────────────────┤
│  dbHandlers.ts (변경 없음)           │
│    └→ postgres.ts                   │
└─────────────────────────────────────┘
```
